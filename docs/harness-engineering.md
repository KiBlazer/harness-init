# Harness Engineering

Harness Engineering uses a project-level control layer to make AI-assisted engineering more reliable and repeatable.

A `.harness/` directory captures:

- stable rules
- project context
- domain boundaries
- skill routing
- verification expectations
- per-change audit trails
- safe Autopilot policy

The goal is not to replace human judgment. The goal is to move repeated decisions into reviewed project artifacts so routine local/dev work can close with evidence and risky work stops with a clear blocker report.
